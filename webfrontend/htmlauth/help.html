Canton Smart Soundbar Plugin

Two-way local MQTT integration for the Canton Smart Soundbar 10.

Setup

  1. Enter the soundbar IP address and click Save Configuration.
  2. Power, input, volume and play mode are controlled over FFAA (TCP port 50006).
  3. The MAC field is optional and currently kept only for compatibility in configuration.

MQTT Inbound Commands (broker/Loxone → plugin)

Publish every command payload to exactly one command topic: loxberry/plugin/cantonbar/cmd (or your configured command topic).

PayloadAction
power_onPower on via FFAA
power_offStandby via FFAA
volume_set_40Set volume to 40 (0–100)
volume_upIncrease volume by step (default 5)
volume_downDecrease volume by step
mute_onMute via FFAA (0x0009)
mute_offUnmute via FFAA (0x0009)
mute_toggleToggle mute via FFAA (0x0009)
mode_stereo, mode_movie, mode_musicSet play mode on current input
input_tv, input_dvd, ...Switch using configured source name aliases

Play mode protocol note: implemented over FFAA command 0x0003 (CMD_INPUT_MODE) using payload byte 3: 1=Stereo, 2=Movie, 3=Music.

Volume protocol note: implemented over FFAA command 0x000C (CMD_VOLUME), then published to MQTT as 0-100 percent.

Mute protocol note: implemented over FFAA command 0x0009 (query + set with payload 0/1).
